    • 此教程源于郭霖的《第一行代码–Android》中关于Git的教程，简单实用，故总结一下。

    Git入门

    安装Git

    • Linux
      打开shell界面

      sudo apt-get install git-core

    创建代码仓库

    • 1.配置身份

      git config –global user.name “zxsong”
      git config –global user.email “zxsong13@gmail.com”

    • 2.创建代码仓库
      cd命令进入项目的目录下面，输入如下命令

      git init

      完成创建代码仓库的操作，此时在项目目录下生成一个隐藏的.git文件夹，这个文件夹就是用来记录本地所有的Git操作的，可以通过ls-al命令查看。如果想删除本地仓库，只需删除该文件夹即可。

    提交本地代码

    代码仓库建立完之后就可以提交代码了，使用add和commit命令即可。
    add—用于把想要提交的代码先添加进来
    commit—执行提交操作

    • 1.添加文件，输入如下命令

      git add AndroidManifest.xml

    • 2.添加文件夹，输入如下命令

      git add src

    • 3.添加所有文 件，输入如下命令

      git add .

    • 4.提交代码，输入如下命令

      git commit -m “First commit”

      一定要通过-m参数来加上提交的描述信息，否则是不合法的。

    Git进阶

    查看修改内容

    • 在项目的根目录下输入如下命令

      git status

    • 查看更改的内容

      git diff

    • 查看单个文件的更改

      git diff src/com/example/providertest/MainActivity.java

    撤销未提交的修改

    • 未执行 add 命令的文件的撤销，使用 checkout 命令

      git checkout src/com/example/providertest/MainActivity.java

    • 执行add命令的文件的撤销
      先取消添加，然后才能撤销修改，取消添加使用 reset 命令

      git reset HEAD src/com/example/providertest/MainActivity.java

      此时就可以使用 checkout 命令将修改的内容撤销了。

    查看提交记录

    • 使用 log 命令查看历史提交信息

      git log

    • 如果只想查看其中一条记录，可以在命令中指定该记录的id，并加上 -1 参数，表示我们只想看到一行记录，如果想看到具体修改了什么内容，再加上 -p 参数

      git log 2b51963c26576921c30cb7fe2e2d8c57fc5d1625 -1 -p

    Git高级

    分支的用法

    • 创建分支
      创建一个名为version1.0的分支

      git branch version1.0

    • 查看当前版本库有哪些分支

      git branch -a

    • 切换分支，从master分支切换到version1.0分支上

      git checkout version1.0

    • 合并分支，将version1.0分支上修改并提交的内容合并到master分支上

      git checkout master
      git merge version1.0

    • 删除分支，将version1.0分支删除

      git branch -D version1.0

    与远程版本库协作

    • 将远程版本库克隆到本地

      git clone https://github.com/xuesongzh/MyMall.git

    • 将本地修改的内容同步到远程版本库

      git push origin master

      其中，origin 指定的是远程版本库的Git地址，master 指定的是同步到哪一个分支上

    • 将远程版本库上的修改同步到本地，使用fetch或pull两种命令
      1.fetch命令

      git fetch origin master

      将远程版本库上的代码同步到本地，但不会合并到任何分支上，而是存放到一个origin/masster分支上，可以通过diff命令来查看远程版本库上修改的内容

      git diff origin/master

      再调用merge命令将origin/masster分支上的修改合并到主分支上

      git merge origin/masster

      2.pull命令
      可以从远程版本库上获取最新的代码并合并到本地

      git pull origin master

      Git的常见用法就介绍到这，更详细的讲解请参看Git教程 - 廖雪峰的官方网站
